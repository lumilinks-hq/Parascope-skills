エラーメッセージカタログ After

/zen clarify モード適用後 — 原因 + 対処法の構造で改善

フォームバリデーション — 具体例と対処法を添える

必須フィールド
お名前 お名前を入力してください
改善点: 「必須です」→「お名前を入力してください」。フィールド名を含めて、何が必要か明確にした。
メールアドレスの形式
メールアドレス メールアドレスの形式を確認してください 例: name@example.com
改善点: 「不正な形式です」→「形式を確認してください」+例示。攻撃的な「不正」を排除し、正しい形式を見せて自己解決を促す。
文字数制限
自己紹介 200文字以内 現在 243 文字です。200 文字以内に短くしてください あと 43 文字減らす必要があります
改善点: 「文字数オーバー」→具体的な数値（現在243文字 / 上限200文字 / あと43文字）。ユーザーが対処量を見積もれる。
パスワード確認
パスワード（確認） 上のパスワードと一致しません。もう一度入力してください
改善点: 「一致しません」→「上のパスワードと一致しません」。比較対象を明示し、やり直しを促す文を追加。

HTTPエラー — 導線と次のアクションを用意する

404 Not Found
404
お探しのページが見つかりませんでした
URLが変更されたか、ページが削除された可能性があります
トップページへ 前のページに戻る
改善点: 行き止まりの解消。原因の推測 +「トップへ」「戻る」のリンク + 検索フォームの3段構えで、ユーザーの次のアクションを複数提示。
500 Internal Server Error
500
処理を完了できませんでした
サーバーで一時的な問題が発生しています。
しばらく待ってからもう一度お試しください。
お問い合わせ
改善点: 「Internal Server Error」→「処理を完了できませんでした」。技術用語を排除し、一時的な問題であることを示す。再試行ボタンと問い合わせリンクで回復手段を提供。

権限・認証エラー — 威圧感を排除、具体的な案内へ

403 Forbidden
🔐
このファイルを編集する権限がありません
閲覧権限のみ付与されています。編集が必要な場合は、ワークスペースの管理者に権限の変更を依頼してください。
改善点: 「拒否されました」→「権限がありません」で威圧感を排除。現在の権限レベルを示し、管理者への連絡という具体的な解決手段を提示。
401 Unauthorized
🔑
ログインの有効期限が切れました
セキュリティのため、一定時間操作がない場合は自動的にログアウトされます。
改善点: 「認証エラー」→「ログインの有効期限が切れました」。原因を明確にし、理由（セキュリティのため）も添える。再ログインボタンで即座に対処可能。

通信・接続エラー — 対処ステップと再試行を提供する

リクエストタイムアウト
⏳
応答に時間がかかっています
サーバーからの応答がありませんでした。入力内容は保持されています。
改善点: 「タイムアウトしました」→「応答に時間がかかっています」で状況を平易に伝える。入力内容の保持を明示して安心感を与え、再送信と下書き保存の2つの選択肢を提示。
ネットワーク接続
📶
インターネットに接続できません
ネットワーク接続を確認してください。
  • Wi-Fi またはモバイルデータがオンになっているか確認する
  • 機内モードがオフになっているか確認する
  • ルーターを再起動する
改善点: 「接続エラー」→「インターネットに接続できません」+具体的な対処リスト。ユーザーが自分で問題を特定・解決できるステップを提示。

検索結果・空状態 — 次のアクションとガイドを添える

検索結果ゼロ件
🔍
Figma プラグイン の検索結果はありません
別のキーワードで試してみてください
  • キーワードを短くする（例:「Figma」だけで検索）
  • スペルや表記ゆれを確認する（例:「プラグイン」「plugin」）
  • カテゴリーから探す
改善点: 「結果がありません」→ 検索キーワードを表示して「何を検索したか」を確認可能に。加えて、検索の改善提案を3つ提示して再試行を促す。
空のリスト
📝
まだプロジェクトがありません
最初のプロジェクトを作成して、チームで
タスク管理を始めましょう。
改善点: 「データがありません」→「まだプロジェクトがありません」。文脈に合わせた文言 + 最初のアクションを促すCTAボタン。空の状態をオンボーディングの機会に変える。
